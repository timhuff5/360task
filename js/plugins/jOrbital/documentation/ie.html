Internet explorer

jOrbital will work for IE version 8 or above.

Reason I've disabled it for IE 6,7 is that planets were clipped by their orbits.

But even in IE8...

It will not work without a little hack. IE is ignoring transparent backgrounds for mouse-over events. This behaviour causes to close orbits right after mouse pointer leaves the jOrbital object.

Solution is very simple. You have to set an transparent image ( gif or png ) for .orbit class

.orbit { background:url(transparent.gif) }

I've made an transparent gif and it's included somewhere in documentation. Filename is ietrans.gif

Span vs DIV

Sometimes jOrbital will not work with inline elements like span on IE. Even if you specify display:block, so it's recommended to use elements that are blocks by default.
